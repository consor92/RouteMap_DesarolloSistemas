4Análisis Funcional y Requisitos

🎯 Conceptos Clave

  • ¿Qué son los requisitos funcionales?
  • ¿Qué son los requisitos no funcionales?
  • Técnicas para identificar requisitos
  • Priorización de requisitos (MoSCoW)
  • Listado de funcionalidades
  • Especificación de requisitos
  • Trazabilidad de requisitos

📚 Recursos para Aprender

  • Requisitos Funcionales vs No Funcionales - YouTube
  • Ingeniería de Software - Sommerville (Capítulos 4-5)
  • Gestión de Requisitos - Atlassian
  • Método MoSCoW - HubSpot
  • Requisitos Funcionales - Lucidchart

💪 Ejercicios Prácticos

  1. Análisis del relevamiento: Revisar la información recolectada e identificar necesidades
  2. Lista de funcionalidades: Crear una lista completa de todas las funcionalidades del sistema
  3. Requisitos funcionales: Documentar cada funcionalidad como requisito funcional
  4. Requisitos no funcionales: Identificar requisitos de rendimiento, seguridad, usabilidad, etc.
  5. Priorización MoSCoW: Clasificar requisitos en Must have, Should have, Could have, Won't have
  6. Documento de requisitos: Crear documento formal con todos los requisitos
🎯 Ejemplo de Requisitos para Sistema Educativo:
  • RF001: El sistema debe permitir el registro de nuevos estudiantes
  • RF002: El sistema debe mostrar información institucional en la página principal
  • RNF001: El sistema debe cargar en menos de 3 segundos
  • RNF002: El sistema debe ser responsive para móviles

📊 Evaluación

Rúbrica - Análisis de Requisitos

Especificación Requisitos claros, medibles y verificables Requisitos bien especificados Especificación básica pero comprensible Especificación ambigua o incompleta
Priorización Priorización lógica y bien justificada Priorización adecuada Priorización básica Priorización confusa o sin justificar
